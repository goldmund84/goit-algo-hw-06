# goit-algo-hw-06

## Завдання 1 — модель транспортної мережі

- Побудовано неорієнтований граф для міст Київ, Житомир, Вінниця, Львів, Луцьк, Люблін, Варшава та Краків (`task-1.py`, дані — `transport_network.py`).
- Ребра графа відповідають автомагістралям між містами з вагою у кілометрах.
- Програма рахує кількість вершин і ребер, середній ступінь, а також визначає міста з максимумом/мінімумом зв’язків.
- За наявності `matplotlib` генерується зображення `task-1-network.png` з підписами міст і довжинами маршрутів.

## Завдання 2 — пошук шляхів DFS і BFS

- Скрипт `task-2.py` підключає той самий граф і будує шляхи з Києва до Кракова за допомогою двох алгоритмів:
  - **DFS** (глибини): `Kyiv -> Zhytomyr -> Vinnytsia -> Lviv -> Lublin -> Warsaw -> Krakow`.
  - **BFS** (ширини): `Kyiv -> Lutsk -> Lviv -> Krakow`.
- Різниця пояснюється стратегією обходу:
  - DFS рухається найглибшою гілкою графа, тому спершу заходить у Житомир–Вінницю і робить більше кроків, перш ніж дістатися до Кракова.
  - BFS розширює пошук рівнями, тож одразу знаходить сусіда другого рівня — Луцьк, через якого найкоротший маршрут веде до Львова й Кракова.
- Порівняня обох результатів виводиться у консоль разом з поясненням, чому саме такий шлях знайшов кожен алгоритм.

## Завдання 3 — алгоритм Дейкстри

- У `task-3.py` реалізовано власну версію алгоритму Дейкстри на базі черги з пріоритетами.
- Запуск програми обчислює найкоротші шляхи між усіма парами міст у транспортному графі та друкує їх у кілометрах разом з маршрутами.
- Ваги ребер (довжини маршрутів), додані ще в першому завданні, використовуються як вартість переходів між вершинами.

## Запуск

```bash
cd goit-algo-hw-06
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python3 task-1.py   # аналітика та зображення мережі
python3 task-2.py   # порівняння DFS і BFS
python3 task-3.py   # найкороткі шляхи за Дейкстрою
```
